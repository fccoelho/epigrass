522      To run a function in a separate thread, simply put it in a Future: 
523   
524      >>> A=Future(longRunningFunction, arg1, arg2 ...) 
525   
526      It will continue on its merry way until you need the result of your function.  
527      You can read the result by calling the Future like a function, for example: 
528   
529      >>> print A() 
530   
531      If the Future has completed executing, the call returns immediately.  
532      If it is still running, then the call blocks until the function completes.  
533      The result of the function is stored in the Future, so subsequent calls to  
534      it return immediately. 
535   
536      A few caveats: 
537      Since one wouldn't expect to be able to change the result of a function,  
538      Futures are not meant to be mutable. This is enforced by requiring the
